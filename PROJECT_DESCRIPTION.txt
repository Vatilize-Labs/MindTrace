EXECUTIVE SUMMARY
================================================================================

MindTrace is a decentralized 1v1 strategic deduction game built on Stellar
Soroban that demonstrates how truly confidential multiplayer interactions can
exist on blockchain networks without central authority.

The game is a proof-of-concept for private, trustless competitive gaming where:
- Game states are cryptographically secured
- Player wagers are locked and distributed fairly
- Outcomes are verified transparently by smart contracts
- No central authority controls the competition

This represents a new category of Web3 experiences combining game theory,
cryptography, and economic incentives to create fair, private, multiplayer
entertainment on decentralized networks.

================================================================================
CORE GAMEPLAY MECHANICS
================================================================================

THE GAME
--------
A 1v1 strategic deduction game where:
- One player (Creator) encodes a hidden binary choice (0 or 1)
- The other player (Challenger) must deduce the correct answer
- Both players wager equal amounts as "skin in the game"
- Smart contracts verify answers and distribute the winner's pot

THE CONCEPT
-----------
Players are presented with a scenario (e.g., "Where is Alex hiding?")
- Option 0: "In the weeds"
- Option 1: "Behind the building"

The Creator knows where Alex is and encodes this secretly on-chain.
The Challenger must use logic, intuition, and strategic reasoning to guess
correctly.

WINNING CONDITIONS
------------------
1. Challenger guesses correctly
   -> Winner: Challenger
   -> Reward: Full pot (both wagers combined)

2. Challenger guesses incorrectly
   -> Winner: Creator
   -> Reward: Full pot (clever deception reward)

3. Creator doesn't reveal in time
   -> Winner: Challenger
   -> Reward: Full pot (timeout protection)

FAIRNESS MECHANISMS
-------------------
- Smart contract enforcement: All rules enforced without human judgment
- Timeout protection: Creator must reveal within 10,000 blocks (~83 min),
  else Challenger wins
- No reneging: Both players commit funds before gameplay; early exit has
  penalties
- Transparent outcomes: Winner determination is cryptographically verifiable

================================================================================
KEY FEATURES & INNOVATIONS
================================================================================

1. PRIVATE, TRUSTLESS GAMEPLAY

Instead of trusting a gaming platform or central authority, trust is replaced
with:
- Cryptographic verification
- Immutable smart contract logic
- Transparent outcome determination
- Automatic fund distribution

Benefit: Players can engage with confidence that no platform can cheat them.

2. ECONOMIC INCENTIVE STRUCTURE

Wager mechanism prevents:
- Frivolous gameplay (players have skin in the game)
- Spam attacks (each game costs participants something)
- Rage quitting (players can renege but lose funds)
- Abuse vectors (economic disincentives)

Benefit: Only serious players participate, ensuring quality matchups.

3. MULTIPARTY PRIVACY MODEL

Demonstrates how multiple participants can interact within shared on-chain
state while maintaining control over information visibility:
- Creator's answer is hidden until revelation
- Challenger's guess is revealed only for scoring
- Both players see each other's addresses but not sensitive data
- All state transitions are transparent but informationally private

Benefit: Players have guarantees of information security without central
authority managing that security.

4. SCALABLE ECONOMICS

On Stellar Soroban:
- Per-game cost: ~$0.002 USD (21,000 stroops)
- Cost reduction: 50-100x cheaper than Aleo implementation
- Accessibility: Enables micropayment gaming models
- Practical: Accessible to players with small stakes

Benefit: True mass-market blockchain gaming becomes viable.

TECHNOLOGY ARCHITECTURE
================================================================================

SMART CONTRACTS (SOROBAN/RUST)
------------------------------
Language:           Rust + Soroban SDK
Compilation:        WebAssembly (WASM)
Binary Size:        12 KB (optimized)
Contract Name:      mindtrace_soroban.wasm

Core Functions:
1. propose_game()           - Creator initiates game with wager
2. submit_wager()           - Challenger matches wager and answer
3. accept_game()            - Transition to active gameplay
4. reveal_answer()          - Smart contract compares answers
5. finish_game()            - Distribute pot to winner
6. finish_game_by_timeout() - Auto-resolve after timeout
7. challenger_renege_stake() - Creator early exit with refund
8. opponent_renege_stake()   - Mutual refund before accept
9. get_game()               - Query game state

State Management:
- Game struct: stores all game data (players, answers, wagers, state)
- Game states: PROPOSED → WAGER_MATCHED → ACCEPTED → REVEALED → FINISHED
- Events: published for off-chain listeners (frontend, analytics)

FRONTEND (REACT/TYPESCRIPT)
---------------------------
Framework:          React 18.2
Language:           TypeScript
Build Tool:         Vite 4.5 (dev server: localhost:4000)
State Management:   Zustand
Routing:            React Router v6
Styling:            TailwindCSS
Wallet Integration: Freighter (Stellar wallet)

Key Components:
- Welcome Screen - Account connection & authentication
- Home Dashboard - Active games & game history
- New Game Flow - Multi-step game creation
- Gameplay UI - Real-time game state display
- Account Management - Balance, address display, disconnect

GAME STATES & LIFECYCLE
================================================================================

STATE DIAGRAM
-------------
PROPOSED (1)
    ↓
WAGER_MATCHED (2)
    ↓
ACCEPTED (3)
    ↓
REVEALED (4)
    ↓
FINISHED (5) ← Winner claims pot

ALTERNATE PATHS:
    ├─ RENEGED (0) ← Early exit with refund
    └─ TIMEOUT (6) ← Auto-resolve after ~83 min

STATE DESCRIPTIONS
------------------
PROPOSED
- Creator locked wager, awaiting Challenger
- Duration: Until Challenger acts

WAGER_MATCHED
- Both locked amounts, submitted answers
- Duration: Until accepts or reneges

ACCEPTED
- Game active/binding, no more reneging
- Duration: Until reveal or timeout

REVEALED
- Answers compared, winner determined
- Duration: Until finish_game() called

FINISHED
- Pot distributed, game complete
- Duration: Game over

RENEGED
- Early exit with refund
- Duration: Game terminated

TIMEOUT
- 10,000+ blocks passed, Challenger wins
- Duration: Game over

GETTING STARTED
---------------
1. Install Freighter: https://www.freighter.app/

2. Clone repository:
   git clone https://github.com/Vatilize-Labs/MindTrace.git
   cd MindTrace

3. Install dependencies:
   pnpm install

4. Start frontend:
   npm run dev
   (Visit http://localhost:4000)

5. Deploy contract:
   ./deploy.sh

6. Configure environment:
   - Add CONTRACT_ID from deployment to .env.local

7. Connect wallet and play!
   - Click "Connect Wallet"
   - Approve Freighter connection
   - Create a new game!
